# Пространство Wemade

Рабочее Next.js-приложение операционной системы агентства: внутреннее пространство команды и клиентские кабинеты в одном продукте.

## Что работает

- Переключение ролей: команда, руководители, новичок и клиент.
- Разделение внутреннего и клиентского контуров.
- Клиент видит только свои проекты, клиентские документы, события и обсуждения.
- CRUD для проектов, задач, документов и базы знаний.
- Архивирование, поиск, фильтр по статусу, сортировка, история изменений.
- Демо-режим с локальным сохранением в `localStorage`.
- Production-режим с Supabase Auth, PostgreSQL, RLS и общей базой данных.
- Конструктор сайта для редактирования бренда, главного экрана, цветов, страниц и включенных разделов.
- Подготовлены Supabase schema, Storage-ready поле документов и RLS-политики.

## Запуск локально

```bash
pnpm install
pnpm dev
```

Откройте `http://localhost:3000`.

## Supabase

1. Создайте проект Supabase.
2. Скопируйте `.env.example` в `.env.local`.
3. Заполните `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
4. Поставьте `NEXT_PUBLIC_USE_MOCK_API=false`.
5. Выполните SQL из `supabase/migrations/001_schema.sql`, затем `002_demo_seed.sql`, затем `003_site_builder.sql`, затем `004_production_auth_storage.sql`.
6. Следуйте инструкции в `PRODUCTION.md`.

## Как редактировать сайт без кода

Внутри приложения выберите внутреннего пользователя, например `Лена · Опердиректор`, и откройте раздел `Конструктор`.

Там можно менять:

- название продукта и подпись в меню;
- буквы логотипа;
- заголовок и текст первого экрана;
- основные цвета;
- включение и выключение разделов;
- страницы, адреса, аудиторию и тексты.

В локальной версии эти настройки сохраняются в браузере. В production-режиме они сохраняются в Supabase, чтобы все пользователи видели одну и ту же актуальную версию.

## Деплой на Vercel

1. Подключите репозиторий к Vercel.
2. Добавьте переменные окружения из `.env.example`.
3. Убедитесь, что миграции выполнены в Supabase.
4. Запустите обычный Vercel deploy.
